📁 Agentic Folder Automation (Local, Zero API-Token Loop)
"Select / Create Agent Folder" needs one click (browser security requirement for showDirectoryPicker). After that, everything below — reading files, running inference, writing outputs, renaming processed files — runs with no further clicks, driven by an interval loop. Reloading the page will require one more click to re-confirm folder permission; that's a browser-enforced limit, not something JavaScript can route around.

File variables: write ${input}, ${filename}, or ${generation} anywhere in the prompt above to reference the file currently being processed. Any other ${name} is looked up against the other files sitting in the folder right now, matched by the variable name shown in the File Browser panel below — e.g. ${sensor1} pulls in the live content of sensor1.in.txt. This substitution happens in JavaScript before anything is sent to the model, so a prompt like Combine ${sensor1} and ${sensor2}, then explain: ${input} works even though the model itself never sees the placeholder syntax. If a referenced file is missing, it's marked [MISSING FILE VARIABLE: ...] instead of silently disappearing.

Convention: drop somename.in.txt files into the selected folder (generation 1). Each pass finds unprocessed *.in.txt files, resolves the template, and streams the reply live into the terminal above just like a manual Stream click. Clean model output goes to somename.gN.out.txt; the original input plus a commentary block (prompt template, variables resolved, chain status) goes to somename.gN.done.txt; and — if auto-chain is on and the generation cap hasn't been hit — the output is copied into somename.g(N+1).in.txt so the next pass keeps refining it. Compile & Instantiate must be run first so myActiveConversation exists.
